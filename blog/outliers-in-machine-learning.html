Understanding Outliers in Machine Learning

February 15, 2024 · 15 min read

In the realm of machine learning, outliers represent more than just anomalous data points. They often carry crucial information about edge cases, system failures, or unique patterns that might be valuable for our models.

What Defines an Outlier?

The classical definition of an outlier often relies on statistical measures. Consider a data point that lies beyond three standard deviations from the mean: 

import numpy as np

def identify_outliers(data, threshold=3):
    mean = np.mean(data)
    std = np.std(data)
    z_scores = np.abs((data - mean) / std)
    return z_scores > threshold

Robust Methods for Handling Outliers

Instead of simply removing outliers, we might want to use robust statistical methods. Here's an implementation of the Huber loss function, which is less sensitive to outliers: 

def huber_loss(y_true, y_pred, delta=1.0):
    error = y_true - y_pred
    is_small_error = np.abs(error) <= delta
    squared_loss = 0.5 * np.square(error)
    linear_loss = delta * np.abs(error) - 0.5 * np.square(delta)
    return np.where(is_small_error, squared_loss, linear_loss)

"The art of handling outliers lies not in their elimination, but in understanding their nature and choosing the appropriate method to deal with them."

Practical Considerations

When dealing with outliers in production systems, we need to consider several factors:

  • Domain knowledge and business context
  • Data collection process and potential errors
  • Impact on model interpretability
  • Computational efficiency of detection methods
